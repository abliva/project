# Visio UML 图自动生成系统 v2.0

## 项目简介

Visio UML 图自动生成系统是一款基于 Python 和 Microsoft Visio 的智能化统一建模语言（UML）图表自动构建工具。该系统通过调用 Visio COM 接口实现自动化绘图，能够快速生成专业的用例图、类图和对象图，大幅提升软件设计文档的编写效率。

## 核心功能

### 1. 支持的图表类型

#### 📋 用例图（Use Case Diagram）
- 自动绘制参与者（Actor/火柴人）
- 创建用例椭圆并关联到参与者
- 支持系统边界框的自动生成
- 智能布局算法优化图形位置
- 包含关系（Include）与扩展关系（Extend）

#### 📦 类图（Class Diagram）
- 标准的三层类框结构（类名、属性、方法）
- 支持访问权限修饰符（+、-、#）
- 接口与实现关系
- 泛化（继承）、关联、依赖、聚合、组合关系
- 多重性标注支持

#### 🎯 对象图（Object Diagram）
- 对象实例的可视化表示
- 属性值的显示
- 对象间的链接关系
- 基于类图的实例化展示

### 2. 智能特性

✅ **智能布局引擎**：采用启发式算法自动计算最优图形位置  
✅ **标准UML模具**：优先使用 Visio 内置 UML 静态结构模具确保规范性  
✅ **容错机制**：当标准模具不可用时自动回退到基础绘制方案  
✅ **多路径搜索**：自动在多个可能位置查找模具文件  
✅ **颜色编码**：支持自定义形状填充色增强可读性  
✅ **连接线管理**：智能处理形状间的各种UML关系连线  

### 3. Visio 集成能力

- **完整 COM 自动化**：通过 pywin32 实现对 Visio 的全面控制
- **多版本兼容**：支持 Visio 2016 及更高版本
- **文档格式输出**：保存为标准 .vsdx 格式（Visio 2013+）
- **页面设置**：自定义页面大小、方向和缩放比例
- **模具管理**：自动加载和管理 UML 相关模具库

### 关键技术实现

#### 1. Visio COM 自动化
```python
# 初始化 Visio 应用
self.visio_app = win32com.client.Dispatch("Visio.Application")
self.visio_app.Visible = True

# 创建新文档
self.document = self.visio_app.Documents.Add("")
self.page = self.document.Pages.Item(1)
```

#### 2. UML 形状创建流程
1. 加载 UML Static Structure 模具文件 (`UMLSTNC.VSS`)
2. 从模具中获取 Master 对象（如 Actor、Use Case、Class）
3. 使用 `page.Drop()` 方法将形状放置到指定坐标
4. 设置形状文本内容和样式属性
5. 使用连接器建立形状间的关系

#### 3. 智能回退机制
当标准 UML 模具不可用时：
- **参与者**: 程序化绘制火柴人（圆形头部 + 线条身体）
- **用例**: 绘制椭圆形状
- **类**: 绘制矩形并分割为三个区域
- **接口**: 绘制矩形 + 接口标识

## 使用方法

### 基本使用流程

1. **启动程序**
```bash
python ai画用例图等图.py
```

2. **选择图表类型**
   - 程序会显示交互式菜单
   - 输入对应数字选择要生成的图表类型

3. **输入图表数据**
   - 根据提示输入参与者名称
   - 输入用例/类名及其描述
   - 定义它们之间的关系

4. **自动生成与保存**
   - 系统自动调用 Visio 进行绘图
   - 完成后保存为 .vsdx 文件
   - 可选择保存路径和文件名

### 示例：生成用例图

```python
from ai画用例图等图 import *

with VisioAutomation() as visio:
    factory = VisioShapeFactory(visio)
    
    # 创建参与者
    actor = factory.create_uml_shape('actor', 2, 5, '用户')
    
    # 创建用例
    usecase1 = factory.create_uml_shape('usecase', 5, 5, '登录系统')
    usecase2 = factory.create_uml_shape('usecase', 5, 3.5, '查看数据')
    
    # 建立关联
    visio.connect_shapes(actor, usecase1)
    visio.connect_shapes(actor, usecase2)
    
    # 保存文档
    visio.save_document('output/use_case_diagram.vsdx')
```

## 支持的 UML 元素

### 形状元素
| 元素 | 英文名称 | 说明 |
|------|---------|------|
| 参与者 | Actor | 表示系统外部用户或实体 |
| 用例 | Use Case | 描述系统功能 |
| 类 | Class | 定义对象的属性和方法 |
| 接口 | Interface | 定义契约和规范 |
| 包 | Package | 组织相关元素 |

### 关系类型
| 关系 | 符号 | 用途 |
|------|------|------|
| 关联 | 实线 | 对象间的一般关系 |
| 泛化 | 空心三角箭头 | 继承关系 |
| 实现 | 虚线空心三角 | 接口实现 |
| 依赖 | 虚线箭头 | 使用关系 |
| 聚合 | 空心菱形 | 整体-部分（弱拥有） |
| 组合 | 实心菱形 | 整体-部分（强拥有） |
| 包含 | 虚线箭头 <<include>> | 必须执行 |
| 扩展 | 虚线箭头 <<extend>> | 可选执行 |

## 常见问题解决

### Q: 提示"无法启动 Visio"
**A**: 
- 确认 Visio 已正确安装
- 尝试以管理员身份运行程序
- 检查 Visio 是否已激活

### Q: 找不到 UML 模具文件
**A**: 
- 系统会自动搜索多个可能的安装路径
- 如果找不到，会使用基础绘制方式作为替代
- 不影响最终图表的正确性

### Q: 生成的图表布局不理想
**A**: 
- 可以手动在 Visio 中调整位置
- 或使用 Visio 的"自动布局"功能
- 未来版本将增强布局算法
